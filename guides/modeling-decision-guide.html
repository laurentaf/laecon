Modeling Decision Guide

Econometrics + ML — a reference card for the working economist. Flow: basic → advanced.

01

OLS Regression

When to Use

Continuous outcome. Exploring linear relationships, causal inference (with identification strategy), baseline for comparison.

Key Assumptions

  • Linearity in parameters
  • Exogeneity: E[ε|X] = 0
  • Homoscedasticity, no autocorrelation
  • Full rank, normal errors (for inference)

Python

import statsmodels.api as sm model = sm.OLS(y, X).fit( cov_type='HC1' # robust SE ) print(model.summary())

statsmodels fit_intercept cov_type

Diagnostics

R² Explained variance (0–1)
F-test Joint significance of regressors
Breusch-Pagan Heteroscedasticity (p < 0.05 → reject H₀ of homoscedasticity)
VIF Multicollinearity (> 10 = concern)
02

GLS / WLS / Robust SE

When to Use

OLS estimates are unbiased but standard errors are wrong: heteroscedasticity (cross-section), autocorrelation (time series), or unequal precision across observations.

Key Assumptions

  • Mean function is correct (same as OLS)
  • Variance structure is misspecified under OLS
  • GLS: full knowledge of Ω (variance matrix)
  • WLS: weights proportional to 1/Var(εᵢ)

Python

import statsmodels.api as sm # Robust SE (Huber-White) m_robust = sm.OLS(y, X).fit(cov_type='HC1') # HAC (Newey-West for autocorrelation) m_hac = sm.OLS(y, X).fit( cov_type='HAC', cov_kwds={'maxlags': 4} ) # WLS weights = 1 / X_var m_wls = sm.WLS(y, X, weights=weights).fit() # GLS m_gls = sm.GLS(y, X, sigma=Omega).fit()

HC1 HC3 HAC Newey-West

Diagnostics

White test General heteroscedasticity
BP test Breusch-Pagan for form
DW statistic First-order autocorrelation
LM test Serial correlation (Breusch-Godfrey)

Decision: if BP/White rejects → switch to HC1/HC3. If DW shows autocorrelation → HAC/Newey-West. WLS only when you know the variance function.

03

Logit / Probit

When to Use

Binary outcome (0/1). Logit gives odds ratios; Probit gives latent-index interpretation. Logit is default; Probit when you need normal CDF link.

Key Assumptions

  • Binary dependent variable
  • Independence of irrelevant alternatives (IIA) for logit
  • No perfect separation
  • Adequate sample size (rule of 10 events per covariate)

Python

import statsmodels.api as sm # Logit m_logit = sm.Logit(y, X).fit() print(m_logit.summary()) # Probit m_probit = sm.Probit(y, X).fit() # Marginal effects (crucial for interpretation) mfx = m_logit.get_margeff(at='overall') print(mfx.summary())

odds ratios marginal effects predict()

Diagnostics

Pseudo R² McFadden's (0.2–0.4 = good fit)
LRT Likelihood ratio test (overall significance)
Hosmer-Lemeshow Calibration (p > 0.05 = good)
AUC-ROC Discrimination (> 0.8 = good)

Always report marginal effects, not just coefficients. Logit coefficients are not directly interpretable.

04

Ordered Logit / Probit

When to Use

Ordinal outcome with meaningful order but unequal intervals. Classic case: NPS (0–10 scale), Likert scales, credit ratings (AAA → D).

Key Assumptions

  • Outcome is ordinal (ordered categories)
  • Proportional odds assumption: effects are constant across all cutpoints
  • Latent variable is continuous + logistic/normal
  • Categories are correctly ordered

Python

from statsmodels.miscmodels.ordinal_model import OrderedModel # Ordered Logit m_ologit = OrderedModel(y, X, distr='logit').fit() # Ordered Probit m_oprobit = OrderedModel(y, X, distr='probit').fit() # Cutpoints are estimated, not specified print(m_ologit.summary())

cutpoints proportional odds brant test

Diagnostics

Brant test Proportional odds (p > 0.05 = assumption holds)
LRT Overall model significance
Predicted probabilities Per category
Cutpoint monotonicity Cutpoints must increase

If Brant test fails → consider generalized ordered logit (partial proportional odds) or collapse categories.

05

Grouped Logit

When to Use

Aggregated binary data. E.g., NPS by region: each row is a region with count of promoters/detractors. Much faster than individual-level logit when data is summarized.

Key Assumptions

  • Data is grouped: success count + total count per row
  • Binomial distribution within groups
  • Independence across groups
  • Variance = n·p·(1−p) per group

Python

import statsmodels.api as sm # y = [successes, failures] per group # X = group-level predictors y_grouped = np.column_stack([nps_promoters, nps_detractors]) m = sm.GLM( y_grouped, X, family=sm.families.Binomial() ).fit() print(m.summary())

GLM Binomial weighted aggregate

Diagnostics

Residual deviance Over/under-dispersion
Pearson χ² Goodness of fit
AIC/BIC Model comparison
Deviance residuals Per-group fit

If overdispersion detected → use quasi-binomial family or switch to beta-binomial.

06

Poisson / Negative Binomial

When to Use

Count data (non-negative integers: number of claims, visits, defaults). Poisson is the baseline; Negative Binomial when overdispersion is present (variance > mean).

Key Assumptions

  • Poisson: mean = variance (equidispersion)
  • Neg. Binomial: variance = μ + αμ² (allows overdispersion)
  • Counts are independent
  • Log link: log(E[y]) = Xβ → multiplicative effects

Python

import statsmodels.api as sm # Poisson m_pois = sm.GLM( y, X, family=sm.families.Poisson() ).fit() # Negative Binomial m_nb = sm.GLM( y, X, family=sm.families.NegativeBinomial() ).fit() # Or via discrete model m_nb2 = sm.NegativeBinomial(y, X).fit()

GLM log link incidence rate ratio

Diagnostics

Cameron-Trivedi Overdispersion test
Likelihood ratio Poisson vs NegBin (α = 0?)
PEARSON χ²/df Dispersion ratio (≈1 = good)
Vuong test Poisson vs NegBin comparison

Exponentiated coefficients = Incidence Rate Ratios (IRR). IRR = 1.05 → 5% increase in expected count per unit increase in X.

07

Panel Models (FE / RE)

When to Use

Cross-sectional time series: multiple units observed over time. Individual effects (heterogeneity) need to be absorbed. Classic: firm-level data, country panels, household longitudinal.

Key Assumptions

  • FE: unit effects correlate with regressors (absorbs all time-invariant heterogeneity)
  • RE: unit effects are random/uncorrelated with regressors
  • Exogeneity conditional on unit effects
  • No lagged dependent variable in FE (Nickell bias)

Python

from linearmodels.panel import PanelOLS, RandomEffects # Set multi-index: (entity, time) df = df.set_index(['firm', 'year']) # Fixed Effects fe = PanelOLS(y, X, entity_effects=True, time_effects=True).fit() # Random Effects re = RandomEffects(y, X).fit() # Hausman test from linearmodels.panel import compare print(compare({'FE': fe, 'RE': re}))

linearmodels entity_effects time_effects

Diagnostics

Hausman test FE vs RE (p < 0.05 → use FE)
F-test Entity effects significance
Wooldridge Serial correlation in FE
Cluster SE Robust to within-group correlation

Default: FE is safer (no correlation assumption). RE only when you need to estimate time-invariant effects or predict random intercepts.

08

Time Series (ARIMA / VAR)

When to Use

Single or multiple time series. ARIMA for univariate forecasting; VAR for multivariate dynamics. Cointegration for long-run equilibrium relationships.

Key Assumptions

  • Stationarity (or cointegrated if non-stationary)
  • ARIMA: (p,d,q) orders capture dynamics
  • VAR: all variables are endogenous
  • Cointegration: long-run equilibrium + short-run adjustment

Python

from statsmodels.tsa.arima.model import ARIMA from statsmodels.tsa.api import VAR from statsmodels.tsa.stattools import adfuller # Stationarity test adf = adfuller(y) # ARIMA(p,d,q) m_arima = ARIMA(y, order=(1,1,1)).fit() # VAR model = VAR(df[['y1','y2','y3']]) results = model.fit(maxlags=4) print(results.summary())

ADF test Johansen Granger causality IRF

Diagnostics

ADF / KPSS Unit root tests
ACF / PACF Identify p, q orders
Johansen Cointegration rank
Ljung-Box Residual autocorrelation

Pipeline: (1) test stationarity → (2) difference if needed → (3) identify orders via ACF/PACF → (4) estimate → (5) check residuals white noise.

09

Decision Tree / Random Forest / XGBoost

When to Use

Non-linear relationships, high-dimensional data, interactions without specification. Prediction-first problems. When interpretability is secondary to accuracy — but SHAP is mandatory to open the black box.

Key Assumptions

  • No linearity assumption (strength + weakness)
  • Trees handle interactions automatically
  • RF: bagging reduces variance
  • XGBoost: boosting reduces bias (sequential correction)

Python

from sklearn.ensemble import RandomForestClassifier import xgboost as xgb import shap # Random Forest rf = RandomForestClassifier( n_estimators=500, max_depth=8, min_samples_leaf=20, random_state=42 ).fit(X_train, y_train) # XGBoost xgb_model = xgb.XGBClassifier( n_estimators=300, learning_rate=0.05, max_depth=6, subsample=0.8 ).fit(X_train, y_train) # SHAP — MANDATORY explainer = shap.TreeExplainer(xgb_model) shap_values = explainer.shap_values(X_test) shap.summary_plot(shap_values, X_test)

SHAP required sklearn xgboost

Diagnostics

SHAP values Feature importance (per-prediction)
Cross-validation Out-of-sample performance
Learning curves Overfitting / underfitting
Permutation importance Robustness check

Rules: (1) Never deploy without SHAP summary + dependence plots. (2) Compare against a simple econometric baseline. (3) Report both in-sample and out-of-sample metrics.

10

Model Selection

When to Use

Every time. Model selection is not a step — it's a mindset. You're always comparing at least two candidates with different assumptions, complexity levels, and error trade-offs.

Information Criteria

  • AIC: -2ln(L) + 2k. Prediction-oriented. Favors larger models.
  • BIC: -2ln(L) + k·ln(n). Consistent. Favors parsimony.
  • HQIC: compromise between AIC and BIC.
  • Lower is always better. Differences > 10 = substantial evidence.

Python

import statsmodels.api as sm # Compare models m1 = sm.OLS(y, X1).fit() m2 = sm.OLS(y, X2).fit() print(f"AIC: {m1.aic:.1f} vs {m2.aic:.1f}") print(f"BIC: {m1.bic:.1f} vs {m2.bic:.1f}") # Likelihood ratio test from scipy.stats import chi2 lr = 2 * (m2.llf - m1.llf) df = m2.df_model - m1.df_model p = chi2.sf(lr, df) # K-fold cross-validation from sklearn.model_selection import cross_val_score scores = cross_val_score(model, X, y, cv=5, scoring='neg_mean_squared_error')

AIC BIC LRT k-fold CV

When to Switch Models

Misspecification RESET test fails → try log-transform or tree
Heteroscedasticity BP rejects → robust SE or WLS
Non-linearity Residuals show pattern → polynomial or tree
Overfitting CV gap widens → regularize (LASSO, Ridge, shrink trees)

Philosophy: start simple (OLS), justify complexity with evidence. The economist's default is parsimony; the ML engineer's default is accuracy. You need both.
